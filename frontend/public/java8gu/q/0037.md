# ✅从 innodb 的索引结构分析，为什么索引的 key 长度不能太长?

> 题号：0037 ｜ 分类：05_MySQL

---

## 速记

- InnoDB的索引通常使用的是B+树的数据结构。B+树的结构特性和磁盘I/O操作的特性就决定了索引的key如果太长会影响性能。
- 在上面的文章汇中，介绍过，InnoDB的数据页是InnoDB存储引擎中用于存储数据的基本单位。
- 数据页是磁盘上的一个连续区域，通常大小为16KB当然，也可以通过配置进行调整。
- 还有就是，在B+树中进行键值查找时，如果key长度过长，比较操作的成本会增加，尤其是对于字符串这类可变长度的数据类型。
- 所以，索引的 key 长度不建议太长。
