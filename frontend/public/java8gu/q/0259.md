# ✅JVM的运行时内存区域是怎样的？

> 题号：0259 ｜ 分类：02_JVM

---

## 速记

- 根据Java虚拟机规范的定义，JVM的运行时内存区域主要由Java堆、虚拟机栈、本地方法栈、方法区和程序计数器以及运行时常量池组成。
- 需要注意的是，上面的这6个区域，是虚拟机规范中定义的，但是在具体的实现上，不同的虚拟机，甚至是同一个虚拟机的不同版本，在实现细节上也是有区别的。
- **程序计数器**：一个只读的存储器，用于记录Java虚拟机正在执行的字节码指令的地址。
- **Java虚拟机栈**：一种线程私有的存储器，用于存储Java中的局部变量。
- **本地方法栈**：本地方法栈是一种特殊的栈，它与Java虚拟机栈有着相同的功能，但是它支持本地代码（ Native Code ）的执行。
- **Java堆**：是存储对象实例的运行时内存区域。
- **方法区**：用于存储已被加载的类信息、常量、静态变量、即时编译后的代码等数据的内存区域。
- **运行时常量池**：是方法区的一部分。

### 堆和栈的区别

- 堆和栈是Java程序运行过程中主要存储区域，经常被拿来对比，他们主要有以下区别（这里的栈主要指的是虚拟机栈）：
- 1、**存储位置不同**，堆是在JVM堆内存中分配空间，而栈是在JVM的栈内存中分配空间。
- 2、**存储的内容不同**，堆中主要存储对象，栈中主要存储本地变量
- 3、**堆是线程共享的**，栈是线程独享的。
- 4、堆是垃圾回收的主要区域，不再引用这个对象，会被垃圾回收机制自动回收。栈的内存使用是一种先进后出的机制，栈中的变量会在程序执行完毕后自动释放
- 5、栈的大小比堆要小的多，一般是几百到几千字节
- 6、栈的存储速度比堆快，代码执行效率高
- 7、堆上会发生OutofMemoryError，栈上会发生StackOverflowError

### 什么是堆外内存？

- 堆外内存是指将数据存储在堆以外的内存中，**主要是指将一些直接内存分配在堆以外**，以提高应用程序的性能或节省堆内存空间。
- Java 中的直接内存是通过使用 java.nio 包中的 DirectByteBuffer 类来实现的。

### 方法区的变迁

- 前面我们提过，方法区其实是JVM规范中定义出来的一块区域，具体的虚拟机实现上是有很大的差别的。在不同的版本中，方法区的位置也不尽相同。
