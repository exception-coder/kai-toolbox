# ✅Innodb加索引，这个时候会锁表吗？

> 题号：0920 ｜ 分类：05_MySQL

---

## 速记

- 在 MySQL 5.6 之前，InnoDB 索引构建期间会对表进行排它锁定，这意味着在索引构建期间，其他会话不能读取或修改表中的任何数据。
- **自 MySQL 5.6 开始，InnoDB 使用一种称为“Online DDL”的技术，允许在不阻塞其他会话的情况下创建或删除索引**。
- 从上表中可以看到，当我们创建、删除或者重命名一个索引的时候，是会用到in place的模式。
- 需要注意的是，虽然Online DDL 可以减少锁定时间和对性能的影响，但在索引构建期间仍然可能出现锁定和阻塞。
