# ✅扫表任务，如何写SQL可以避免出现跳页的情况？

> 题号：0264 ｜ 分类：20_任务调度

---

## 速记

### 错误写法（会跳页）

```sql
SELECT * FROM t WHERE state='INIT' ORDER BY id LIMIT 100,100;
```

- 第一页处理完后 state→SUCCESS，原 100~200 行已经"上移"到第一页
- 用 `LIMIT 100,100` 实际取的是第 3 页，第 2 页被跳过
- 极端情况：第一页有失败数据卡住，永远只查第一页 → **死循环**

### 正确写法（游标法）

```sql
SELECT * FROM t WHERE state='INIT' AND id > #{last_max_id}
ORDER BY id LIMIT 100;
```

- 记录上一页最大 id，下一页从该 id 之后开始
- 保证不重不漏
